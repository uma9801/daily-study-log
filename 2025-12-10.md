# 引数
ここで言う引数は、公式の[用語集](https://docs.python.org/ja/3/glossary.html#term-parameter)では仮引数(parameter)と呼ばれる。
>(仮引数) 名前付の実体で [関数](https://docs.python.org/ja/3/glossary.html#term-function) (や [メソッド](https://docs.python.org/ja/3/glossary.html#term-method) ) の定義において関数が受ける [実引数](https://docs.python.org/ja/3/glossary.html#term-argument) を指定します。仮引数には5種類あります:
## kwargs
可変長キーワード引数（var-keyword argments）
- var(variable): 可変
- vary: (…を)(いろいろに)変える、変更する、改める、(…に)変化を添える、(…を)多様にする
`**kwargs` ** （アスタリスク）が2つ付いていれば良い。引数名は変えられる。
例：`**name`
`**kwargs`部分で渡した引数は、関数には**キー：値**の辞書形式で渡される。
渡す引数の数も、随時変えられる。
```python
def game_title(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")
game_title(name="Chrono Trigger", company="SQUARE", price=9800)
```
## args
可変長位置引数（var-positional arguments）
`*args`* は1つ。こちらも引数名は変えられる。
可変長キーワード引数はキー：値の辞書形式だったが、こちらは値のみを引数として渡す。
**位置**というのは、記述した順番（位置）どおりに引数が渡されるということ。
渡される形式は**タプル**。
```python  
def game_lists(*args):
    for value in args:
        print(value)
game_lists("Chrono Trigger", "Final FantasyIV", "聖剣伝説3")
```
他にも以下の仮引数名がある
- 位置またはキーワード引数（デフォルトの仮引数）
	- `def func(foo, bar=None): ...`
- 位置専用
	- `def func(posonly1, posonly2, /, positional_or_keyword): ...`
- キーワード専用
	- `def func(arg, *, kw_only1, kw_only2): ...`
# sqlite3
SQLiteは動的型付け(Dynamic Typing)
PostgreSQLやMySQL、Oracleはテーブル作成時にカラムの型を宣言するが、SQLite(sqlite3)は宣言をせず値の柔軟な受け入れができる。
sqlite3：開発の柔軟性、学習コストの低さがメリット
MySQL等：データの信頼性・堅牢性やバグの見つけやすさ、実行速度の速さ、スケーラビリティがメリット。更に
>静的型付けデータベースはスキーマ（設計図）が明確に定義されているため、他の人が見たときにもデータの構造が理解しやすく、共同作業やメンテナンスが容易になります。
